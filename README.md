# 📘 멋쟁이사자처럼 14기 백엔드 스터디

📌 GitHub 과제 제출 안내

안녕하세요!
백엔드 과제는 GitHub를 통해 제출합니다. 아래 방법을 따라 진행해주세요.

---

## ✅ 1. 과제 제출 방법

1. 저장소 clone

2. weekN 폴더로 이동 (예: week1)

3. 아래 구조로 파일 생성

   weekN/이름/파일

   예시:
   week1/wooseok/Main.java

4. 코드 작성

5. ⚠️ 아래에서 반드시 선택

   * "Create a new branch" 선택

6. 브랜치 이름 입력

   * feature/이름-weekN

7. Commit 진행

8. "Compare & pull request" 클릭

9. PR 생성 → 제출 완료

---

## ✅ 2. 규칙

* main 브랜치에 직접 업로드 금지 ❌
* 반드시 브랜치를 생성하여 작업 ✔
* PR(Pull Request)로 제출 ✔
* 폴더 구조 반드시 준수 ✔

---

## ✅ 3. 브랜치명 규칙

feature/이름-weekN

예시:
feature/wooseok-week1

---

## ✅ 4. PR 제목 규칙

[주차] 이름 과제 제출

예시:
[1주차] 류우석 과제 제출

---

## 💡 용어 설명 (중요🔥)

### 📁 week 폴더

→ 주차별 과제를 넣는 "제출 위치"

예:
week1 = 1주차 과제 폴더

---

### 🌿 브랜치

→ main에 영향을 주지 않고 작업하는 "개인 작업 공간"

예:
feature/wooseok-week1

---

### 🔁 PR (Pull Request)

→ 과제를 제출하는 과정 (제출 버튼 역할)

---

### 📌 main 브랜치

→ 최종 코드가 모이는 곳 (직접 수정 금지)

---

## 💥 핵심 정리

폴더 = 어디에 넣을지
브랜치 = 어떻게 제출할지

👉 과제 제출 = 파일 업로드가 아니라 PR 생성입니다.
